\section{Objectives}

\subsection{Functional Objectives}

\subsubsection{Automatic Hit Detection}

The main goal is to create a software module that can detect the moment a dart hits the target and the section it has hit. It analyzes the video stream, finds a new stationary dart, and maps it to the correct section.

\subsubsection{Interactive Web Interface}

It is necessary to implement a web interface that can be accessed on the same computer that analyzes the video stream. Players can interact with it using other devices connected to the same local network, such as smartphones or tablets.

\subsubsection{Support for Multiple Game Modes}

The developed application supports different versions of the game. Most commonly used versions include 501 and 301. In addition, the following rule variations have been implemented: Double In and Double Out. These versions cover the most popular ones, making it easier to play with the developed software.

\subsection{Technical Objectives}

\subsubsection{Motion-Based Dart Recognition}

The main problem is to recognize the moment a new dart is thrown, distinguishing it from those already in the board and static objects in the background. This is achieved through motion detection. By analyzing the difference between two consecutive frames in a video stream, it is possible to detect the moment a dart hits the target and the point it has come to rest at.

\subsubsection{Manual and Marker-Based Calibration Concepts}

In order to translate the image coordinates into the scoring regions, the need for a calibration process arises. Two methods are discussed for the purpose. First, the manual method of calibration, where the user is required to align the board landmarks such as the bullseye and the ring boundaries, is discussed. Secondly, the marker-based method, where augmented reality markers are placed on the board, is also discussed.
